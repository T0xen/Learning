
# Coding Standards & Definition of Done

## Naming Conventions – 

- Variables – Camel Case 
- Methods – Camel Case
- Constants – All Caps 

## Commenting Conventions – 

- Comments explaining the purpose of a method
- Complex code we can add additional comments for more detail 

## Whitespace and Indentation – 

- Line break after an item before an XML comment for the next item 
- Follow Kernighan & Ritchie (K&R) style 
  - Ex – 
```
Foo(int x, int y) {           
            Return x + y; 
} 
```
## Technological Standards – 

- Development Environment – GitHub, Git 
-	Languages – Java 
-	IDE – Visual Studio Code

## Definition of Done –

-	Test (min: 1 happy/ 2 unhappy path) 
-	Follows coding standard
-	Completes the PBI/Meets acceptance criteria
